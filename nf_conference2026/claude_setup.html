Setting Up Claude Code in VS Code

  1. Go to VS Code and select the Download button in the middle of the screen.

  2. Once the file has finished downloading, open your Downloads folder and double-click on the file VSCode-darwin-arm64.dmg. A pop-up window should appear like the one below (for Mac):

  3. Follow the instructions in the pop-up window. On Mac, drag the Visual Studio Code icon into the Applications folder. Bring up the app by searching for Visual Studio Code in Spotlight Search (on Mac) or Windows Search (on Windows).

  4. Select the Extensions button on the left sidebar where the mouse cursor is on the image above. This will pull up a side window where you can search for claude and install the extension called Claude Code for VS Code as seen in the image below.

  5. Then click the Claude Code button in the top right corner to bring up the extension.

  6. Now you should see something similar to the image below with Claude Code on the right window.

  7. You can clean up the screen by clicking the Extension button on the left sidebar again to minimize the window.

  8. And close the Extension: Claude Code for VS Code window by clicking the X button on the top menu.

  9. Your screen should look similar to below.

  10. In Claude Code, click login with Anthropic Console and allow the pop-up window to open.

  11. This will pull up a login page on your browser like the one below. Login with the email you used to register for the workshop, and follow the login instructions on the screen.

  12. If you are asked which organization to connect, pick NF AI Workshop.

  13. Authorize access, then close the window when prompted and go back to VS Code.

  14. Your Claude Code window should now look like this.

Starting a Project with Claude Code

2.1  Setting up a new project

  1. Now let's start a project. Click the Open... button on the Welcome page.

  2. Make a new folder and name it something related to your project.

  3. And open the folder.

  4. Your VS Code should now look like this:

  5. In the top-right corner, click the ... menu next to CHAT and select Claude Code to pull up Claude in the window.

Now you are good to go! In the section below, we will introduce some basic commands when using VS Code and Claude.

2.2  An introduction to VS Code

VS Code is a popular software development environment.

  1. On the left-hand menu, click the file + icon to create a new file. You can also drag-and-drop existing files to add them to your project.

  2. The folder + icon to the right creates a new folder. Use this to organize your project.

  3. At the top menu bar, select Terminal -> New Terminal (or use the shortcut Ctrl+Shift+tilde) to bring up a new terminal.

  4. The new terminal will appear at the bottom of your screen. You can use it to run scripts. If you're not sure how, feel free to ask Claude which commands to run or let Claude run it for you.

2.3  Claude settings

  1. Type / in the bottom text bar like so, and it will bring up some common settings.

    • Switch model... lets us choose which model to use. The table below gives a short description of the latest models. We recommend Opus 4.8 for coding tasks.

    • Effort determines how many tokens Claude is willing to use when responding to requests. You can read more about it here.
    • Thinking will enhance reasoning capabilities and increase the amount of time Claude takes to produce an answer.

  2. Select the </> button at the bottom of the prompt box to change permission settings. For larger projects, we recommend to use Plan mode first, and then edit automatically once you think the plan looks good.

  3. Click the + button in the top-right corner to start a new chat.

  4. The clock icon in the top-right corner lets you go through previous conversations.

To learn more about using Claude in VS Code, take a look at the documentation.

2.4  Tips on using Claude in VS Code

General tips

  • CLAUDE.md is a file that gives Claude a set of rules to follow throughout the whole project. It automatically loads into every session, so you don't need to re-explain general guidelines. Create a CLAUDE.md at the root of your project and copy-paste this into your file: Karpathy skills for Claude.
  • In the prompt box, you can @-file (e.g. @project_docs.md) to reference a file in your project directly, so Claude doesn't have to go look for it.
  • Manage context-- how much information Claude sees at a time-- by running /clear or /compact <instructions> (e.g. /compact Focus on changes) fairly often to help Claude focus and reduce token usage. Another trick is to let Claude summarize its progress in a PROGRESS.md file as it works so you can use /clear often without losing information about what Claude is doing.
  • Use /btw for quick questions that don't need to be recorded in conversation history.
Visit Claude Best Practices for more tips and tricks on using Claude.

Smaller projects

For smaller projects like personal websites or data filtering scripts, you can use the prompt box to directly tell Claude what to do.

  • use the + button on the bottom left to add relevant files for Claude to reference

You can follow up with any feedback or additional requests by using the prompt box in the same chat.
